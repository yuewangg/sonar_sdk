What's New?

The previous BlueView SDK, version 3 (SDK3) had a few shortcomings:

  • The original SDK was created before multicore CPUs became commonplace. It was unclear which functions were thread-safe under what conditions.
    We are now more explicit about thread safety. The const keyword on an object indicates that the function is thread-safe for that object.


  • Simultaneously building images on more than one thread was impossible without hacky workarounds.
    We added the BVTImageGenerator to create per-thread.

  • The original SDK was created before our micro-bathymetry (MB) sonar were available
    This release offers more accurate image geometry, especially for profile and 3D applications.
  • More sensitive image processing (match filter)
  • It was difficult to tell if certain functions failed.
    All functions which can fail now return an error code.

Migrating from SDK3

Consistent return of error codes

All functions which can fail now return an error code. This means code that looked like:

int ping_count = BVTHead_GetPingCount(head);

Must now be written as:

int count = 0;
int ret = BVTHead_GetPingCount(head, &ping_count);

You are strongly encouraged to use our C++ wrapper or generate your own error wrapping functionality

IF_FAILED_RETURN( BVTHead_GetPingCount(head, &ping_count) );

Ping GetImageXY and GetImageRTheta are obsolete

A Ping is no longer responsible for generating its own images. Instead of:

BVTPing_GetImage(head, &image);

You must now create a BVTImageGenerator and set a Head first:

BVTImageGenerator xyImager = BVTImageGenerator_Create();
BVTImageGenerator_SetHead(xyImage, head); /* once */
BVTImageGenerator_GetImageXY(xyImager, ping &image); /* many times */

Windows library dependencies removed

The bvtutil library and pthread library were removed from the Windows SDK.

  • You no longer need to distribute bvtutil.dll and pthread.dll with your Windows application.

C++ API Changes

The C++ header-only wrapper now throws exceptions. See the C++ wrapper documentation for more information.

C# API Added

We added C# classes that perform all necessary P/Invoke and unmanaged resource cleanup.
