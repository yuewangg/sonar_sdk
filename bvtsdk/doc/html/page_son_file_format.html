.SON File Format

The BlueView .son file format is proprietary. The header of this format holds a configuration data about each head's emitted pulse and transducer geometry, along with calibration data. Some of this data varies per sonar model, while other data is specific to each individual sonar unit. This means the data stored in a BVTPing is insufficient to generate an image. You must use both this SDK and our file format in order to create images.

Warning
There is nothing stopping you from storing images generated by this SDK in your own file format, but be aware that if you bypass our file format and record your images with an incorrect sound speed, then you will not be able to recreate the images with the correct sound speed. The original data will be effectively lost.
